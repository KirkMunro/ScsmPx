An Overview of ScsmPx

The ScsmPx module facilitates automation with Microsoft System Center Service Manager by auto-loading the native modules that are included as part of that product and enabling automatic discovery of the commands that are contained within the native modules. It also includes dozens of complementary commands that are not available out of the box to allow you to do much more with your PowerShell automation efforts using the platform.

Installing the ScsmPx Module the Easy Way

The ScsmPx module is installed in two steps. The first step is to download the installation script, and the second step is to invoke that installation script with parameters identifying where you want the module installed. Note that you can update the ScsmPx module to the latest version at any time simply by performing these steps again.

Rather Drive Stick?

While installation via the installer script is preferred, if your someone who prefers to manually deploy your modules, or if you would like to use the most recent development build of the ScsmPx module, this section is for you. Download the build that you want to install by simply clicking on the folder button at the top of this page that corresponds to the build you are interested in. You should note that the beta release is unsigned, and the official release is signed. Once you have downloaded that file, make sure that it is unblocked by opening its properties and clicking on the Unblock button if it is present. Then extract the contents of that file into any of the following folders:

  • your current user's "Documents\WindowsPowerShell\Modules" folder
  • the "Program Files\WindowsPowerShell\Modules" folder
  • any other folder where you want to install the module

    • Note that it is strongly recommended that the folder you choose be included in the PSModulePath environment variable.

    Once you have the module in place, rename the root folder of the module as "ScsmPx". At this point, manual setup of the module will be complete.
