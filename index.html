An Overview of ScsmPx

The ScsmPx module facilitates automation with Microsoft System Center Service Manager by auto-loading the native modules that are included as part of that product and enabling automatic discovery of the commands that are contained within the native modules. It also includes dozens of complementary commands that are not available out of the box to allow you to do much more with your PowerShell automation efforts using the platform.

Installing the ScsmPx Module the Easy Way

The ScsmPx module is installed with a single PowerShell command. Note that you can update the ScsmPx module to the latest version at any time simply by running this command again.

# If you want to install ScsmPx for all users or update a version already installed # (recommended, requires elevation for new install for all users) & ([scriptblock]::Create((iwr -uri http://tinyurl.com/Install-GitHubHostedModule).Content)) -ModuleName ScsmPx,SnippetPx # If you want to install ScsmPx for the current user & ([scriptblock]::Create((iwr -uri http://tinyurl.com/Install-GitHubHostedModule).Content)) -ModuleName ScsmPx,SnippetPx -Scope CurrentUser

While installation via the online script is preferred, you can also manually install this module if you prefer. Both the ScsmPx module and the SnippetPx module that it is dependent on are hosted on GitHub at github.com/KirkMunro. I strongly recommend using the automated installation method though.

  • your current user's "Documents\WindowsPowerShell\Modules" folder
  • the "Program Files\WindowsPowerShell\Modules" folder (for all users)
  • any other folder where you want to install the module

    • Note that it is strongly recommended that the folder you choose be included in the PSModulePath environment variable. If you are performing automation tasks using ScsmPx, you are required to choose a path that will be visible to the automation tasks' runas account. For automation purposes, the Program Files path is recommended.

    Once you have the module in place, rename the root folder of the module as "ScsmPx". At this point, manual setup of the module will be complete.
